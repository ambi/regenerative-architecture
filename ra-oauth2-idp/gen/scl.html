Vocabulary 94

本仕様で使うドメイン用語の定義。PascalCase が正準名、aliases は実装上の別表記。

ResourceOwner

保護リソースに対するアクセス権限を付与できる実体。本アプリではエンドユーザー。

aliases resource_owner リソースオーナー EndUser
do not confuse with
  • Client — Client はアプリケーション、ResourceOwner は人間。OAuth では明確に区別される。
Client

ResourceOwner に代わって保護リソースへのアクセスを要求するアプリケーション。client_id で識別される。

aliases client クライアント RelyingParty
AuthorizationServer

ResourceOwner を認証し Client にトークンを発行するサーバー。本アプリそのもの。OIDC では IdP と同義。

aliases authorization_server IdP IdentityProvider
ResourceServer

AccessToken を提示する Client に対して保護リソースを返すサーバー。本アプリの外部。

aliases resource_server
Public

client_secret を安全に保持できない Client（SPA, ネイティブアプリ）。PKCE 必須。

aliases public
Confidential

サーバーサイドで client_secret または private_key を保持できる Client。

aliases confidential
AuthorizationCode

短命（60秒以下）の一度限り使用される中間トークン。/authorize で発行され /token で交換される。grant_type 値「authorization_code」と同じ文字列を共有する（RFC 6749 §4.1）。

aliases authorization_code 認可コード
RefreshToken

AccessToken の再発行に使うトークン。提示のたびにローテーションされる。絶対 TTL 30 日。grant_type 値「refresh_token」と同じ文字列を共有する（RFC 6749 §6）。

aliases refresh_token リフレッシュトークン
ClientCredentials

Client 自身が認可主体となる machine-to-machine フロー (RFC 6749 §4.4)。confidential のみ。

aliases client_credentials M2Mグラント
DeviceCode

入力制約のあるデバイス向けのデバイス認可グラント (RFC 8628)。grant_type は URN。

aliases urn:ietf:params:oauth:grant-type:device_code device_code デバイスコード
AccessToken

ResourceServer にアクセスする際に提示するトークン。JWT (PS256 / ES256) として発行、TTL 600秒。

aliases access_token アクセストークン
do not confuse with
  • IdToken — AccessToken は API アクセス用、IdToken は認証情報の表明用。混同してはならない。
IdToken

OIDC が定める、ResourceOwner の認証結果を表明する JWT。iss/sub/aud/exp/iat/auth_time/nonce/azp を含む。

aliases id_token IDトークン
Pkce

Proof Key for Code Exchange (RFC 7636)。code_challenge / code_verifier で認可コード横取り攻撃を防ぐ。本アプリでは例外なく必須。

aliases pkce PKCE
do not confuse with
  • Dpop — PKCE は認可コード保護、DPoP はトークン保護。攻撃面が異なる。
Dpop

Demonstrating Proof of Possession (RFC 9449)。JWK サムプリント (jkt) でトークンを所有鍵にバインドする。

aliases dpop DPoP
Mtls

Mutual-TLS Client Authentication (RFC 8705)。TLS クライアント証明書で Client を認証する。

aliases mtls mTLS tls_client_auth
Par

Pushed Authorization Requests (RFC 9126)。/par に認可リクエストを事前 POST し /authorize は request_uri のみで参照する。FAPI 2.0 必須。

aliases par PAR
