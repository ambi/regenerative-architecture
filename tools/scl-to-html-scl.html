Invariants 9

入力に依らず常に成り立つ不変条件、または決して起きてはならない事象。

presence_preserved

must

入力に存在するセクションは、出力に対応する <section id="..."> を持つ。

target
Render
always (invariant)
∀ S ∈ keys(scl) ∩ SectionKind : output contains '<section id="S">'

absence_preserved

must

入力に存在しないセクションは、出力にも現れない。

target
Render
never (forbidden)
output contains '<section id="S">' where S ∉ keys(scl)

user_text_escaped

must

入力 SCL 由来の文字列は HTML として安全にエスケープされる。

target
Render
never (forbidden)
output contains unescaped '<' '>' '&' or '"' originating from scl values

deterministic

must

同一入力からは常に同一の HTML を返す。時刻・乱数・環境変数に依存しない。

target
Render
always (invariant)
Render(x) ≡ Render(x)

self_contained

must

HtmlDocument は単一ファイルで表示でき、外部資産（ネットワーク / 別ファイル）に依存しない。

target
Render
always (invariant)
rendering HtmlDocument requires no external network resource or sidecar file

navigable_to_each_section

must

閲覧者は HtmlDocument 上の任意の地点から、入力に存在する各 Section に直接遷移できる。

target
Render
always (invariant)
from any point in HtmlDocument, the reader can reach each present Section without leaving the document

cross_references_are_navigable

must

CrossReference は閲覧者がひとつの操作で参照先定義へ遷移できる形で描画される。

target
Render
always (invariant)
∀ CrossReference c : c is presented as a control that navigates the reader to the definition of c

reading_position_is_indicated

must

閲覧者が長い HtmlDocument 上を移動するとき、現在読まれている Section がナビゲーション側で視覚的に区別される。

target
Render
always (invariant)
the Section currently in view is visually distinguished within the navigation

color_scheme_adapts_to_reader

must

HtmlDocument の配色は、閲覧者の環境が示すライト/ダーク設定に追随する。

target
Render
always (invariant)
HtmlDocument's visual presentation follows the reader's light/dark preference

Scenarios 2

受け入れ例。自然文ステップで書かれた振る舞いの具体例。引用された既知の名前は定義へリンクする。

最小の SCL でも HTML になり、無いセクションは出力されない

  1. vocabulary だけを持つ最小の SCL を用意して変換する
  2. 出力に vocabulary セクションが含まれる
  3. 出力に models セクションは現れない

ユーザ文字列の HTML 特殊文字はエスケープされる

  1. definition に特殊文字 <script> を含む SCL を用意して変換する
  2. 出力では特殊文字がエスケープされている
  3. 出力に生の script タグは現れない
