Scenarios 5

受け入れ例。自然文ステップで書かれた振る舞いの具体例。引用された既知の名前は定義へリンクする。

最小の SCL でも HTML になり、無いセクションは出力されない

  1. vocabulary だけを持つ最小の SCL を用意する
  2. それを Render に渡す
  3. 出力に vocabulary セクションが含まれる
  4. 出力に models セクションは現れない

ユーザ文字列の HTML 特殊文字はエスケープされる

  1. definition に特殊文字 <script> を含む SCL を用意する
  2. それを Render に渡す
  3. 出力では特殊文字がエスケープされている
  4. 出力に生の script タグは現れない

CONCEPTION + ADR がディレクトリにあれば Decisions タブに並ぶ

  1. CONCEPTION.md と ADR-001-*.md を含むディレクトリを用意する
  2. そのディレクトリを LoadSite で読み込み Render に渡す
  3. 出力に Decisions タブが現れる
  4. 出力に CONCEPTION 由来の見出しと ADR-001 のアンカーが含まれる

completion 付きの WorkItem は Work Items タブに 1 つのカードとして並ぶ

  1. completion を持つ work-items/<id>.yaml を用意する
  2. そのディレクトリを LoadSite で読み込み Render に渡す
  3. 出力に Work Items タブが現れる
  4. 出力に work item の id をアンカーに持つカードが 1 つ含まれる
  5. 出力に completion の summary が含まれる

--scl だけ与えれば Decisions / Work Items タブは空表示で生成できる

  1. SCL のみ指定して LoadSite を呼ぶ
  2. decisions と work_items は空配列になる
  3. Render は失敗せず、Decisions / Work Items タブには placeholder が出る
